批量删除分支

Git 分支：使用 dlb 和 drb 批量清理本地与远端分支

在日常开发中，我们频繁创建 Git 分支来进行开发和调试，但时间一长，仓库里可能堆满了废弃的分支，影响查找和管理。

手动删除分支不仅繁琐，还容易误删重要分支。为此，我自定义了 dlb（Delete Local Branch）和 drb（Delete Remote Branch）两个命令，提供交互式选择和多选删除功能，可以一目了然地管理分支，避免混乱，提高效率。

dlb 和 drb 介绍

dlb - 删除本地分支

功能：

  • 删除符合指定前缀的本地分支

  • 排除 master 和 develop 等主干分支，避免误删

  • 提供可选交互模式，让用户手动选择要删除的分支

适用场景：

  • 开发完某个功能或修复后，需要删除临时分支

  • 需要批量清理过时的本地分支

drb - 删除远端分支

功能：

  • 删除符合指定前缀的远端分支

  • 先 git fetch -p 更新远端分支列表，确保数据最新

  • 允许用户从列表中选择要删除的分支

适用场景：

  • 远端分支合并后仍然存在，需要手动清理

  • 避免远端仓库积累过多无用分支，影响代码管理

配置方法

将 dlb 和 drb 代码添加到 ~/.zshrc 文件中，并执行以下命令使其生效：

执行source ~/.zshrc重载配置

使用示例

dlb

执行后，它会列出所有以 feature/ 开头的本地分支（排除 master 和 develop），让你选择要删除的分支。

示例输出：

drb

执行后，它会列出所有以 bugfix/ 开头的远端分支，并让你选择要删除的分支。
